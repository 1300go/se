# 프로젝트 I: 프로젝트 정의서 (미니 드라이브)

### 1. 프로젝트 명칭
* 클라우드 파일 공유 시스템 구축

### 2. 개발 시스템 명칭
* Mini Drive (미니 드라이브)

### 3. 개발 시스템 설명
* 조직 내 협업을 위해 파일 최신 버전을 관리하고 중앙 서버에서 안전하게 파일을 공유 및 저장하는 클라우드 시스템입니다.
 * **주요 해결 과제**:
    * **버전 관리**: 다중 사용자 작업 시 발생하는 최신 파일 식별의 혼선 해결
    * **검색 효율성**: 분산된 저장 경로로 인한 탐색 시간 낭비 방지
    * **보안 강화**: 비공식 경로(이메일/메신저) 공유 시 발생하는 외부 유출 위험 차단

### 4. 핵심 기능 
* 파일 업로드 및 다운로드 : 최대 2GB 이상의 대용량 파일 지원 및 전송 성공률 99.9% 보장 
* 계층형 폴더 생성 및 관리 : 사용자별 독리된 디렉터리 구조 제공 및 이동/삭제 가능
* 보안 및 접근 제어 :
   * 다중 인증 : OTP 및 이메일 인증을 통한 2중 보안 체계 구축
   * 로그 기록 : 파일 접근/수정/삭제 시 실시간 감사 로그 생성
   * 역할 기반 권한 제어 : 사용자 역할에 따른 세분화된 읽기/쓰기 권한 부여
* 파일 검색 기능 : 메타데이터(이름, 날짜, 유형) 기반의 1초 이내 검색 결과 반환
* 파일 버전 관리 : 최소 10개 이상의 이전 버전 보관 및 시점 복구 
### 5. 유사 소프트웨어 분석
* **Google Drive / Dropbox**: 범용적인 클라우드 서비스이나, 본 프로젝트는 조직 내부 보안과 200명 규모의 특정 협업 환경에 최적화된 기능을 제공함.
