# 프로젝트 II: 대상 시스템 품질 요소 추정 보고서

본 보고서는 **ISO/IEC 25010 (SQuaRE)** 표준 모델의 품질 특성을 기반으로, 클라우드 파일 공유 시스템(미니 드라이브)의 요구사항을 분석하여 각 스테이크홀더별 품질 기대치를 추정한 결과입니다.

## 1. 사용자가 원하는 기대치 (External Quality)
사용자는 시스템의 기능 완결성과 실질적인 사용 편의성을 중시합니다.
* **기능 적합성(Functional Suitability)**: 파일 업로드/다운로드, 공유 및 권한 설정 등 핵심 기능이 명세대로 정확히 동작해야 하며, 특히 공유 권한(읽기/편집)이 엄격하게 적용되어야 함. 
* **사용성(Usability)**: 별도의 학습 없이도 드래그 앤 드롭으로 파일을 관리할 수 있어야 하며, 실수로 삭제한 파일을 복구할 수 있는 '휴지통' 기능을 통해 사용자 오류로부터 보호받기를 기대함. 
* ]**성능 효율성(Performance Efficiency)**: 약 200명의 동시 접속 환경에서도 파일 검색 및 다운로드 시 응답 시간을 최적화하여 사용자 체감 지연 시간을 최소화해야 함.

## 2. 개발자 및 유지보수 담당자가 원하는 기대치 (Internal Quality)
개발자는 시스템의 지속 가능성과 코드의 구조적 건전성을 중시합니다.
* **유지보수성(Maintainability)**: 향후 AI 기반 검색이나 공동 편집 기능을 추가하기 용이하도록 모듈화가 되어 있어야 하며, 새로운 파일 형식 뷰어를 추가하기 쉬운 확장 구조를 갖춰야 함. 
* **시험 가능성(Testability)**: 동기화 로직의 레이스 컨디션(Race Condition)이나 복잡한 권한 체계를 검증하기 위해 자동화된 테스트가 용이한 구조여야 함. 
* **이식성(Portability)**: 다양한 웹 브라우저(Chrome, Safari, Edge 등) 환경에서 동일한 사용자 경험을 제공하며 시스템이 안정적으로 동작해야 함. 

## 3. 개발 조직의 관리자가 원하는 기대치 (Sponsor & Process Quality)
관리자는 비즈니스 가치 보호와 조직 차원의 운영 안정성을 중시합니다.
* **보안성(Security)**: 최우선 순위 요소로서, 저장 데이터(At-Rest)와 전송 데이터(In-Transit) 모두를 암호화하고 비정상적인 접근 시도를 탐지할 수 있는 기밀성과 무결성을 확보해야 함. 
* **신뢰성(Reliability)**: 서비스 가용성을 99.99% 이상 유지하여 업무 중단을 방지해야 하며, 서버 장애 시에도 데이터 유실이 없도록 다중화(Replication) 체계를 갖춰야 함. 
* **비용 효율성(Cost Efficiency)**: 무제한 버전 저장이 아닌 효율적인 저장 공간 관리 표준을 수립하여 자원 사용을 최적화하고 비즈니스 이득을 극대화하기를 기대함.

## (1) 사용자가 원하는 기대치
* **신뢰성(Reliability)**: 업로드한 다양한 형태(문서, 이미지, 영상 등)의 업무 파일이 데이터 손실 없이 안전하게 저장되고, 언제든 다시 다운로드할 수 있어야 함.
* **사용 용이성(Usability)**: 특별한 프로그램 설치 없이 웹 브라우저를 통해 직관적으로 파일을 업로드하고 관리할 수 있어야 함.
* **효율성(Efficiency)**: 파일 이름, 업로드 날짜, 유형 등 다양한 조건으로 검색했을 때 결과가 신속하게 표시되어야 함.

## (2) 개발자 혹은 유지보수 담당자가 원하는 기대치
* **확장성 및 유지보수성(Maintainability)**: 향후 공동 편집이나 AI 기반 검색과 같은 새로운 기능을 추가할 수 있도록 시스템 구조가 유연하게 설계되어야 함. 
* **변경 가능성(Modifiability)**: 파일 수정 시 이전 버전을 보관하고 필요시 복원할 수 있는 버전 관리 체계가 정확히 동작해야 함. 

## (3) 개발 조직의 관리자가 원하는 기대치
* **보안성(Security)**: 조직의 중요 데이터를 보호하기 위해 권한 기반의 접근 제어와 데이터 전송 암호화가 철저히 이루어져야 함.
* **성능 및 안정성(Performance)**: 약 200명의 직원이 동시에 시스템을 사용하더라도 속도 저하나 오류 없이 안정적인 서비스를 제공해야 함.
