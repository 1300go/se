### 1. 서론

#### 1.1 문서 목적 및 범위
본 문서는 조직 내 협업 시 발생하는 파일 버전 혼선과 보안 유출 문제를 해결하기 위한 Mini Drive 프로젝트의 요구사항을 정의한다. 시스템의 기능적, 비기능적 요구사항과 인터페이스에 필요한 명세를 기술하여 개발 및 검증의 기준으로 활용한다.

#### 1.2 프로젝트 개요
##### 1.2.1 프로젝트 정의
조직 내부의 보안을 유지하면서 200명 규모의 협업 환경에 최적화된 프라이빗 클라우드 파일 공유 시스템을 구축하는 것을 목표로 한다.

##### 1.2.2 주요 기능 설명
| ID | 설명 |
| :---: | :--- |
| F01 | 대용량 파일 업로드 및 다운로드  |
| F02 | 계층형 폴더 구조 생성 및 사용자별 독립 디렉터리 관리 |
| F03 | OTP 및 이메일을 활용한 다중 인증보안 체계 |
| F04 | 메타데이터 기반 고속 파일 검색 |
| F05 | 파일 버전 관리 및 시점 복구 |

#### 1.3 용어 정의
| 용어 | 설명 |
| :---: | :--- |
| OTP | One-Time Password. 본인 확인을 위해 생성되는 일회용 비밀번호 |
| 감사 로그 | 시스템 내에서 발생한 파일 접근 및 수정 행위를 기록한 보안 증적 |
| 메타데이터 | 파일 이름, 생성일, 크기 등 데이터의 속성을 설명하는 핵심 정보 |

### 2. 요구사항

#### 2.1 기능적 요구사항
##### 2.1.1 파일 관리 기능
| 분류 | 요구사항 | 우선순위 |
| :---: | :--- | :---: |
| FR-001 | 사용자는 최대 2GB 이상의 파일을 안정적으로 업로드 및 다운로드한다. | 상 |
| FR-002 | 사용자는 계층형 폴더를 생성하고 파일을 이동 또는 삭제할 수 있다. | 상 |

##### 2.1.2 보안 및 인증 기능
| 분류 | 요구사항 | 우선순위 |
| :---: | :--- | :---: |
| FR-003 | 시스템은 로그인 시 OTP 또는 이메일을 통한 2중 인증을 수행한다. | 상 |
| FR-004 | 관리자는 사용자 역할별로 세분화된 파일 접근 권한을 설정한다. | 상 |
| FR-005 | 시스템은 파일의 접근 및 수정 이력을 감사 로그로 실시간 기록한다. | 중 |

##### 2.1.3 검색 및 버전 관리 기능
| 분류 | 요구사항 | 우선순위 |
| :---: | :--- | :---: |
| FR-006 | 사용자는 파일명, 날짜 등 메타데이터 기반으로 1초 이내에 검색한다. | 상 |
| FR-007 | 시스템은 파일 수정 시 이전 버전을 보관하며 원하는 시점으로 복구한다. | 상 |

#### 2.2 비기능적 요구사항

| 분류 | 요구사항 | 우선순위 |
| :---: | :--- | :---: |
| NFR-001 | 시스템은 Ubuntu 22.04 LTS 및 Windows 환경에서 동작해야 한다. | 상 |

##### 2) 성능 요구사항
| 분류 | 요구사항 | 우선순위 |
| :---: | :--- | :---: |
| NFR-003 | 시스템은 200명 동시 접속 시에도 검색 결과를 1초 이내에 반환한다. | 상 |
| NFR-004 | 시스템은 파일 전송 성공률 99.9% 및 서비스 가용성 99.99%를 보장한다. | 상 |

##### 3) 보안 요구사항
