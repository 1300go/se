## 1. 서론

### 1.1 문서의 목적 및 범위
본 문서는 'Mini Drive' 프로젝트의 전반적인 수행 계획을 정의하며, 효율적인 일정 관리와 품질 보증을 위한 지침서로 활용한다.

### 1.2 프로젝트 개요

#### 1.2.1 프로젝트 정의
조직 내 팀원들이 프로젝트 진행 시 발생하는 파일 버전 혼선과 보안 유출 문제를 해결하기 위한 '프라이빗 클라우드 파일 공유 시스템' 구축을 목표로 한다. 200명 규모의 조직 환경에 최적화된 파일 중앙 관리 기능을 제공한다.

#### 1.2.2 주요 기능 설명
| ID | 기능명 | 설명 |
| :--- | :--- | :--- |
| F01 | 대용량 파일 전송 | 최대 2GB 이상의 파일을 안정적으로 업로드/다운로드 지원 |
| F02 | 계층형 권한 관리 | 사용자별 독립 디렉터리 제공 및 폴더별 접근 권한 제어 |
| F03 | 다중 인증 보안 | OTP 및 이메일 2중 인증을 통한 보안 강화 |
| F04 | 스마트 검색 | 메타데이터 기반으로 1초 이내 원하는 파일 검색 가능 |
| F05 | 버전 관리 | 수정 시 이전 버전을 10개 이상 보관하여 즉시 복구 지원 |

### 1.3 프로젝트 목표
* **실시간 버전 관리 및 복구**: 데이터 손상 시 즉각적인 시점 복구 지원
* **고속 파일 검색 시스템**: 200명 규모 환경에서 1초 이내 검색 가능
* **강력한 다중 보안 체계**: OTP/이메일 인증으로 외부 유출 원천 차단
* **사용자 친화적 UI**: 별도 교육 없이 5분 이내 모든 기능 숙달 가능

### 1.4 용어 정의
* **프라이빗 클라우드**: 특정 조직 내부 구성원만 사용하는 폐쇄형 클라우드 저장소
* **버전 관리**: 파일 수정 이력을 보관하여 과거 시점으로 되돌리는 기술
* **메타데이터**: 파일명, 날짜 등 데이터를 설명하는 핵심 정보
* **감사 로그**: 시스템 내 파일 접근 및 수정 내역을 기록한 보안 증적

### 1.5 참조 문서
* 소프트웨어 공학 강의 자료 (제5장 프로젝트 관리)
* 프로젝트 관리 계획서 샘플 
* Mini Drive 프로젝트 정의서 및 품질 요소 추정 보고서

## 2. 개발 계획

### 2.1 개발 절차 모형
#### 2.1.1 소프트웨어 개발 절차 모형
위험 분석을 반복하며 시스템을 발전시키는 **나선형 모형**을 채택한다. 대용량 전송 및 보안 취약점 등 기술적 리스크를 사전에 파악하여 제거하기 위함이다.

#### 2.1.2 분석·설계 방법론
시스템 재사용성과 유지보수성을 높이기 위해 **객체지향 방법론**을 적용하며, UML을 활용하여 클래스 및 시퀀스 다이어그램을 설계한다.

### 2.2 WBS (Work Breakdown Structure)
| 대분류 | 중분류 | 세부 활동 |
| :--- | :--- | :--- |
| **요구분석** | 계획 수립 | 프로젝트 범위 정의 및 요구사항 명세서 작성 |
| **설계** | 시스템 설계 | 서버 아키텍처, DB 스키마 및 보안 로직 설계 |
| **코딩** | 핵심 기능 구현 | 대용량 업로드/다운로드 및 버전 관리 개발 |
| **테스트** | 검증 및 평가 | 200명 동시 접속 부하 테스트 및 보안 점검 |
| **마무리** | 최종 문서화 | GitHub 소스 정리 및 운영 매뉴얼 작성 |
